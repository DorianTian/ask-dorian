# Ask Dorian — UI Redesign: Fragment-First Design

> **范围**: Showcase UI 全面重设计 — 导航、Today、Weekly、多端、双主题
> **日期**: 2026-03-10
> **状态**: 已批准

## 一、设计动机

当前 showcase 设计存在以下核心问题：

1. **像 Todo List** — 排版以时间线和任务列表为主，看不出和 Todoist/Sunsama 的差异
2. **碎片处理不可见** — Ask Dorian 的核心链路（碎片输入 → AI 理解 → 结构化执行）在 UI 上没有体现
3. **导航臃肿** — 侧边栏 10 个入口、4 个分组，信息过载
4. **主题混搭** — 暗色 AI 面板 + 亮色主区域，视觉割裂
5. **Desktop 定位模糊** — 只设计了 menubar popup，缺乏完整客户端定义

## 二、设计原则

- **Fragment-First**: 碎片流是主角，不是任务清单。用户看到的是"AI 为我做了什么"
- **AI 过程可见**: 每条碎片展示完整处理链路 — 原始输入 → AI 理解 → 实体提取 → 执行动作
- **主题统一**: Light/Dark 两套完整主题，不混搭，主题内视觉一致
- **三端一致叙事**: Web/Desktop/Mobile 同一个碎片处理故事，不同详细度

## 三、导航重设计

### 移除侧边栏，改为顶部水平导航

```
┌──────────────────────────────────────────────────────────────┐
│ ✦ Dorian   [今日] [本周] [项目]         收集箱(3) 回顾  ⌘K │
└──────────────────────────────────────────────────────────────┘
```

| 区域 | 内容 | 说明 |
|------|------|------|
| 左侧 Logo | ✦ Dorian | 品牌标识 |
| 主导航（3 个） | 今日 / 本周 / 项目 | 高频页面，tab 样式 |
| 次要导航（2 个） | 收集箱（带 badge）/ 回顾 | 较低频 |
| 工具 | ⌘K 命令面板 | 知识库/日历/设置/通知/AI 技能 等低频功能入口 |

### ⌘K 命令面板覆盖的功能

通过 ⌘K 访问，不再占用导航栏位：
- 知识库搜索
- 日历视图
- 设置
- 通知中心
- AI 技能
- 全局搜索（碎片/任务/项目/知识）

## 四、Today 页面 — Fragment Feed

### 布局

```
┌─────────────────────────────────────┬──────────────────┐
│           Fragment Feed             │   Compact Panel  │
│                                     │                  │
│  [碎片输入框 — 文字/语音/截图/粘贴]   │  📊 今日概览     │
│                                     │  (碎片数/待确认/  │
│  ✦ Dorian 处理记录                   │   自动执行/事项)  │
│                                     │                  │
│  ┌─ Fragment Card ──────────────┐   │  ⚡ 待决策       │
│  │ 🎤 原始输入（语音/截图/文字）  │   │  (冲突/模糊意图)  │
│  │                              │   │                  │
│  │ ┌─ AI 理解结果 ────────────┐ │   │  📅 紧凑时间线   │
│  │ │ ✦ AI 已理解 · 92%       │ │   │  09:00 ● 晨会    │
│  │ │ 📌任务 👤人物 📅时间 📁项目│ │   │  ── NOW 11:20 ── │
│  │ │ → 将创建 P1 任务         │ │   │  12:30 ● OKR ✦   │
│  │ │ [✓确认] [✎编辑] [忽略]   │ │   │  14:00 ● 1:1     │
│  │ └────────────────────────┘ │   │                  │
│  └────────────────────────────┘   │                  │
│                                     │                  │
│  ┌─ Fragment Card (已处理) ─────┐   │                  │
│  │ 💬 "OKR周报今天写完"    ✓已处理│   │                  │
│  │ → AI 安排到 12:30 空档       │   │                  │
│  └────────────────────────────┘   │                  │
└─────────────────────────────────────┴──────────────────┘
```

### Fragment Card 结构

每张卡片展示一条碎片的完整处理故事：

1. **原始输入** — 来源图标（🎤语音/📸截图/💬文字/📋剪贴板）+ 原文 + 输入渠道和时间
2. **AI 理解结果** — 置信度 + 提取的结构化实体标签：
   - `📌 任务` (indigo)
   - `📅 事件` (green)
   - `👤 人物` (orange)
   - `📁 项目` (purple)
   - `🕐 时间` (blue)
   - `📝 知识` (gray)
3. **执行动作** — AI 将做什么（创建任务/日历事件/存入知识库）
4. **操作按钮** — 确认 / 编辑 / 忽略

### Fragment 状态

| 状态 | 视觉表现 |
|------|----------|
| 处理中 | 紫色 shimmer 进度条，实体逐步浮现 |
| 待确认 | 完整卡片 + 操作按钮 |
| 已自动执行 | 降低透明度，绿色 ✓ 标记 |

### 右侧紧凑面板

- **今日概览** — 4 格统计（碎片已处理/待确认/自动执行/今日事项）
- **待决策** — 时间冲突、模糊意图等需要人工介入的项目（橙色 border-left）
- **紧凑时间线** — 时间 + dot + 名称的极简列表，NOW 指示线，AI 安排标 ✦

## 五、Weekly 页面 — 四象限

```
┌─────────────────────────┬─────────────────────────┐
│   🎯 本周焦点            │   📅 时间分配            │
│   AI 挑选的最重要任务     │   7 天紧凑日历           │
│   (上半 = AI 已安排)      │   (每天几个色块表示事项)  │
├─────────────────────────┼─────────────────────────┤
│   ⏳ 待你决策            │   📊 本周进度            │
│   未安排任务/待处理碎片    │   完成率/专注时数        │
│   /冲突 (下半=需你参与)   │   /AI 洞察              │
└─────────────────────────┴─────────────────────────┘
```

| 象限 | 内容 | 定位 |
|------|------|------|
| 🎯 本周焦点 | AI 从所有任务中挑选的 Top 优先项 | AI 已安排 |
| 📅 时间分配 | 紧凑 7 天视图，色块表示事件/任务密度 | AI 已安排 |
| ⏳ 待你决策 | 未安排的任务、待处理碎片、时间冲突 | 需要你参与 |
| 📊 本周进度 | 完成率、专注时数、AI insights | 回顾性 |

**上半部分 = AI 已为你安排好的，下半部分 = 需要你参与决策的。**

## 七、多端设计

### 三端定位

| 端 | 主界面 | 快捷入口 | 技术栈 |
|----|--------|----------|--------|
| Web | 完整 Fragment Feed + 全功能 | — | Next.js |
| Desktop | 复用 Web 布局 + 原生标题栏 | Menubar popup | Tauri |
| Mobile | 轻量 Today + Fragment Feed | 通知/Share Sheet | React Native |

### Desktop 独占能力（6 项）

Web 做不到的系统级能力，是客户端存在的理由：

1. **Menubar 常驻 + 快捷捕获** — 不开主窗口，从菜单栏直接输入碎片（⌘⇧D）
2. **应用上下文感知** — 通过 Accessibility API 知道当前活跃 App，碎片自动关联项目
3. **智能剪贴板监听** — 复制的内容 AI 自动判断是否值得捕获
4. **系统级专注模式** — 联动 macOS Focus API，屏蔽干扰 App，只放行相关工具
5. **可交互原生通知** — 通知栏直接确认/编辑 AI 结果，不用打开 App
6. **全局 ⌘K** — 任何 App 内唤起搜索+捕获+快速操作

### Mobile 设计

- 顶部问候 + 碎片待确认数
- 碎片输入框（文字 + 语音 + 截图）
- Fragment 卡片（待确认的碎片，带确认/编辑按钮）
- 今日概览统计
- 底部 3 Tab：今日 / 捕获 / 收集箱
- 独占：Share Sheet 捕获（iOS/macOS）

| 能力 | Web | Desktop | Mobile |

| Fragment Feed 完整视图 | ✓ | ✓ 复用 Web | 轻量版 |
| 本周/项目/回顾 | ✓ | ✓ 复用 Web | — |
| Menubar 常驻 | — | ✓ 独占 | — |
| 应用上下文感知 | — | ✓ 独占 | — |
| 智能剪贴板监听 | — | ✓ 独占 | — |
| 系统级专注模式 | — | ✓ 独占 | — |
| 全局 ⌘K | 页面内 | ✓ 全局 | — |
| 可交互通知 | 浏览器通知 | ✓ 系统通知 | ✓ 推送 |
| 语音输入 | Web API | 系统麦克风 | 原生 |
